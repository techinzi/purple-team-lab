# Exercise 04 — Ingress Tool Transfer via Certutil

## Objective

Execute a file download using `certutil.exe` to simulate a
Living-Off-the-Land Binary (LOLBin) technique commonly used by
attackers to transfer tools onto compromised systems. Capture
Sysmon and Windows Event Log telemetry, validate detection
coverage in Wazuh, identify gaps in existing detection rules,
create custom detection (if required), and document findings.

This Atomic Red Team test uses the native Windows utility **certutil.exe** with
the `-urlcache` option to download a file from a remote URL and save it locally.
It simulates an attacker transferring a tool or payload onto the compromised system.

**Attacker objective:**

additional tools, malware, or scripts from remote servers without relying on
external download utilities, helping them blend in with legitimate Windows activity.

## Part 4 — Detection Engineering

### Existing Wazuh Detection

Wazuh includes a generic built-in rule
**92032 — Suspicious Windows cmd shell execution**, which detects the
parent `cmd.exe` process when it is executed with the `/C` parameter.

Although this rule detects the Atomic test execution, it does not specifically
identify the use of `certutil.exe` for downloading a file from a remote location.

---

### Detection Gaps

A dedicated detection should identify the execution of `certutil.exe` with the
`-urlcache` option, which is commonly used by attackers to download payloads
from remote locations.

---

### Custom Wazuh Detection

#### Step 1 — Check for Existing certutil.exe Rules

Before creating a custom detection rule, check whether Wazuh already includes
rules for **certutil.exe**.

On **ubuntu-server01**, run:

```bash
sudo grep -R -n -i "certutil" /var/ossec/ruleset/rules
```

The output shows that Wazuh already contains several rules related to `certutil.exe`:

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/14-create-detection.png)

Reviewing these rules shows that they detect behaviors such as:
- Masqueraded `certutil.exe`
- Decoding files using `certutil.exe`
- PowerShell invoking `certutil.exe` for decoding

However, none of the existing rules detect the use of `certutil.exe` with the
`-urlcache` option to download a file from a remote location.

Since there is no existing low-level rule that detects the execution of
`certutil.exe` for file download, there is no suitable built-in parent
rule (`<if_sid>`) that can be reused.

Therefore, the custom detection will use the generic
**Sysmon Process Creation** group as its parent:

```xml
<if_group>sysmon_event1</if_group>
```

This allows the custom rule to evaluate all Sysmon Event ID 1 process
creation events and specifically detect `certutil.exe` executions using
the `-urlcache` option.

#### Step 2 — Navigate to Rules Editor

In Wazuh Dashboard, navigate to: `Left sidebar > Server management > Rules`

Click on **Manage rules files**:

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/15-create-detection.png)

Click on **Custom rules**:

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/16-create-detection.png)

Click the **Edit** (pencil) icon for `local_rules.xml`:

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/17-create-detection.png)

The editor opens with the contents of `local_rules.xml`.

#### Step 3 — Create the Custom Detection Rule

Add the following rule inside `local_rules.xml` and click **Save**:

```xml
<group name="sysmon,command_and_control,T1105,">
    <rule id="100220" level="10">
        <if_group>sysmon_event1</if_group>
        <field name="win.eventdata.image" type="pcre2">(?i)certutil\.exe$</field>
        <field name="win.eventdata.commandLine" type="pcre2">(?i)-urlcache</field>
        <description>T1105 - File Download via certutil urlcache</description>
        <mitre><id>T1105</id></mitre>
    </rule>
</group>
```

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/18-create-detection.png)

Now click on **Restart**, confirm the restart when prompted and wait for
the Wazuh Manager to restart.

#### Step 4 — Verify Rule is Loaded

After the manager restarts, go to Wazuh Dashboard and navigate to:
`Left sidebar > Server management > Rules`

Search rule ID **100220** in search bar. If the rule appears in the results,
it has been loaded successfully.

![Image](/images/atomic-red-team-exercises/04-certutil-download-t1105/19-create-detection.png)
